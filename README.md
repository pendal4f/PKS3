# PKS3 — Мониторинг HTTP‑запросов (WPF)

Учебное WPF‑приложение из практики №3: в одном окне есть сервер, клиент и мониторинг.

## Что умеет

### Сервер (HttpListener)
- Адрес: `http://localhost:<порт>/`
- `GET /` → JSON со статусом (uptime, счётчики, среднее время)
- `POST /` с `{ "message": "..." }` → `{ "id": "..." }`
- Запросы обрабатываются параллельно (многопоточность)

### Клиент (HttpClient)
- `GET` на внешний API
- `POST` на локальный сервер с JSON
- Ответ показывается в приложении

### Логи и аналитика
- Таблица логов + фильтры (метод, `2xx/4xx/5xx`)
- Текстовый лог снизу на вкладке “Сервер”
- Сохранение в `logs.txt` рядом с exe
- График нагрузки (минуты/часы)

## Интерфейс (где что находится)

В приложении две вкладки:

### “Сервер”
- Слева: порт, кнопка запуска, статистика, график нагрузки
- Справа: таблица логов
- Снизу: текстовый лог
- Разделители (`GridSplitter`) позволяют растянуть логи
- Ссылка `http://localhost:...` кликабельна (открывается в браузере)

### “Клиент”
- URL, метод `GET/POST`, JSON‑тело, кнопка “Отправить”
- “Ответ” показывает результат
- “Подсказки” — примеры запросов

## Где править

- UI: `PKS3\MainWindow.xaml`, `PKS3\MainWindow.xaml.cs`
- Сервер: `PKS3\Services\HttpServerService.cs`
- Клиент: `PKS3\Services\HttpClientService.cs`
- Тема: `PKS3\Themes\Dark.xaml`
